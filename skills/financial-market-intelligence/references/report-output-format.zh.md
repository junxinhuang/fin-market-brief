# 日报输出格式

## 目标

金融日报的信息密度较高，默认应生成 HTML 完整版，使用咨询报告 / 投研简报风格展示。飞书消息用于提醒、摘要和跳转，不应承载全部复杂表格。

## 最终完全体默认规则

每日金融日报默认就是“最终完全体”，不是试验版、精简版或临时版。

除非用户明确要求做减法，否则：

- 不得删除既有模块。
- 不得因为飞书长度、移动端展示或某个数据源失败而压缩 HTML 完整版。
- 不得在标题或头部写“版本”“验证版”“新增”等过程性文字。
- 报告标题固定保持专业中性，例如“金融市场跟踪日报”。
- 头部只保留数据截止时间、覆盖资产/数据源等必要信息。
- 英文缩写首次出现必须带中文解释，后文高频缩写也尽量采用“英文(中文)”或“中文(英文)”形式。
- 必须包含术语解释表，至少覆盖 BJT、BLS、FOMC、SEP、VWAP、funding、OI、ETF、spread 等当日报告用到的术语。
- 涉及“防守、相对防御、弱修复、盘前修复、转中性、确认反转”等操作语言时，必须写清楚实际操作含义。
- 所有资产默认使用“最近一个已经确认有数据的有效交易/报价点”。不得为了早报/晚报时段强行使用当日刚开盘、盘前、盘后、周末或假期的空字段。
- 早报中的 A 股默认使用最近有效收盘/报价；晚报中的美股默认使用最近有效收盘/报价。若确实使用盘中/盘前字段，必须确认字段非空且在报告中标明。

## 默认输出

每次完整日报应同时准备：

1. **HTML 完整版**
   - 用于阅读、归档、截图、打印或导出 PDF。
   - 必须保留政策与数据日历、今日核心资产、回溯、转向提醒、持续事项和资产下钻。

2. **飞书摘要版**
   - 用于快速提醒。
   - 必须包含一句话结论、重点转向、今日核心操作含义和 HTML 报告位置。
   - 若没有可访问的外部链接，应说明完整报告已在 Codex 线程或本地工作区生成。

## 文件归档与链接规则

日报文件必须不可变归档。禁止用今天的内容覆盖昨天或更早日期的 HTML 文件。

默认路径：

```text
reports/daily/YYYY-MM-DD-HHMM-market-brief.html
```

规则：

- 每次生成日报、重推日报或盘中更新，都必须创建一个新的 HTML 文件。
- 文件名必须包含报告日期和时间，例如 `2026-06-10-0900-market-brief.html`。
- 不得把 6 月 9 日内容写进 6 月 8 日链接，也不得用新版内容覆盖旧版同名文件。
- 如果同一天多次生成，使用不同时间戳区分，例如 `2026-06-09-0900-market-brief.html`、`2026-06-09-1822-market-brief.html`。
- 飞书每次推送必须发送本次新文件的固定链接，而不是复用旧链接。
- 可以维护 `reports/daily/latest.html` 或首页索引指向最新版，但 `latest.html` 只能作为最新入口，不能作为历史日报链接。
- 回溯分析必须读取历史文件或历史记录，而不是假设旧链接仍保存旧内容。
- 若用户要求“重新生成某日历史报告”，也应新建修订文件，例如 `YYYY-MM-DD-HHMM-market-brief-revision.html`，并在报告中说明修订对象和原因。

GitHub Pages 链接对应规则：

```text
https://junxinhuang.github.io/fin-market-brief/reports/daily/YYYY-MM-DD-HHMM-market-brief.html
```

飞书摘要必须包含该次报告的唯一链接。

## HTML 版式

参考模板：

```text
reports/templates/daily-market-brief.html
```

HTML 必须包含以下模块；这些模块默认不得减少：

- 报告标题、数据截止时间、数据源说明。
- 一句话结论。
- 3-5 个核心驱动。
- 重点转向提醒。
- 昨日判断回溯。
- 政策与数据日历。
- 今日核心资产。
- 术语解释。
- 加密专项。
- 美股与黄金专项。
- A 股专项：仅在当前报告范围包含 A 股时保留；若用户已要求移除 A 股，不得自动加回。
- 持续重要事项。
- 观察清单 / Watchlist。
- 来源和数据缺口说明。

以下模块如果在模板或前一日报中已经存在，也必须保留，除非用户明确要求删除：

- 跨资产仪表盘。
- 美股专项。
- 加密专项。
- 黄金专项。
- 核心资产下钻。
- 机会与风险提醒。
- 操作语言解释。

全量模块参考 `benchmark-inspired-modules.zh.md`。在用户明确要求做减法前，日报采用加法版结构。

## 英文和术语规则

英文可保留，但必须降低理解成本：

- `BJT` 写作 `北京时间(BJT)`。
- `BLS` 写作 `BLS(美国劳工统计局)`。
- `FOMC + SEP` 写作 `FOMC + SEP(美联储议息会议 + 经济预测摘要)`。
- `CPI` 写作 `CPI(消费者价格指数)`。
- `PPI` 写作 `PPI(生产者价格指数)`。
- `PCE` 写作 `PCE(个人消费支出价格指数)`。
- `GDP` 写作 `GDP(国内生产总值)`。
- `VWAP` 写作 `成交量加权均价(VWAP)`。
- `funding` 写作 `资金费率(funding)`。
- `OI` 写作 `未平仓合约(OI)`。
- `ETF` 写作 `ETF(交易所交易基金)`。
- `spread` 写作 `买卖价差(spread)`。
- `risk-on` 写作 `风险偏好(risk-on)`。
- `stablecoin supply` 写作 `稳定币总供给` 或 `链上美元流动性`。

资产和代码可保留英文，但首次出现要补含义：

- `BTC(比特币)`、`ETH(以太坊)`、`SOL(Solana)`。
- `SPY(标普500 ETF)`、`QQQ(纳斯达克100 ETF)`、`SMH(半导体 ETF)`、`GLD(黄金 ETF)`、`TLT(长期美债 ETF)`、`UUP(美元指数 ETF)`。

## 加密专项规则

加密专项不得只写 BTC/ETH/SOL 价格和 funding。默认必须包含：

- BTC/ETH/SOL 当前价格、1h/4h VWAP、funding、OI、24h volume。
- 恐惧贪婪指数。
- 稳定币总供给、1日/7日/30日变化。
- 链上美元流动性判断：扩张、收缩或中性。
- 当前价格结构与链上流动性的交叉结论。

结论写法：

- 价格站上 4h VWAP 且稳定币 7日扩张：可写“修复质量改善”。
- 价格站上 4h VWAP 但稳定币 7日收缩：必须写“价格修复但链上购买力收缩，反弹质量打折”。
- funding/OI 不拥挤但稳定币收缩：写“杠杆不拥挤，但缺少中期购买力确认”。

不得把 Binance/Farside/Coinalyze/Coinglass 失败或超时写入分析正文；只允许在“来源和数据缺口”说明。

## A 股专项规则

A 股专项只在当前报告范围包含 A 股时启用；若当前日报已经按用户要求移除 A 股，应跳过本节，不得因为模板历史内容自动恢复。

启用 A 股时，必须保留板块判断，并新增代表个股层。

默认板块与代表股篮子：

- 科技 / 半导体 / AI：中芯国际、寒武纪、海光信息、中科曙光、工业富联、北方华创、立讯精密。
- 红利 / 高股息：中国神华、长江电力、工商银行、农业银行、江苏银行、中国石化、伊利股份。
- 金融 / 券商：中信证券、国泰君安、海通证券、华泰证券、中国平安、招商银行。
- 地产链 / 消费：万科A、保利发展、美的集团、贵州茅台、五粮液、中国中免。
- 新能源 / 顺周期：宁德时代、隆基绿能、比亚迪、万华化学、宝钢股份、紫金矿业。

输出字段至少包含：

- 板块。
- 代表个股及涨跌幅。
- 当前判断。
- 背景原因。
- 操作含义。
- 失效条件。

数据源规则：

- 优先使用 `skills/financial-market-intelligence/scripts/a_share_sector_snapshot.mjs`。
- 该脚本优先东方财富，失败后使用腾讯公开行情备源。
- 若指数数据源失败但个股数据可用，不得写成 A 股整体“无法刷新”；应区分“指数/资金流待补，个股层已刷新”。
- 早报不要强行取 A 股当日刚开盘字段；如开盘初期字段为空或失真，使用最近有效收盘/报价，并标注“最近有效数据”。

## 设计规则

- 风格应偏投研 dashboard，不做营销页。
- 优先表格、标签、卡片和短段落。
- 颜色用于表达状态：偏空、偏多、警惕、中性，而不是装饰。
- 表格字段不得因为飞书或移动端展示困难而省略。
- 宽表格必须支持横向滚动。
- HTML 应可直接打开，也应适合后续导出 PDF。

## 飞书推送规则

飞书自定义机器人 Webhook 不适合直接渲染任意 HTML。默认策略：

1. 飞书发送摘要。
2. 摘要中说明完整 HTML 报告位置。
3. 如后续接入飞书文件上传、飞书云文档或对象存储，再发送可点击链接。

如果必须在飞书内完整展示，应改用飞书富文本 `post` 或交互卡片，而不是纯 text。
