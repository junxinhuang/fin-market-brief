# 日报连续性与跟踪规则

## 目标

每日金融消息汇总不能是孤立报告，必须串联前几日判断，形成可追踪的投资信息链。

连续性依赖历史文件不可变。每次日报必须创建新的日期时间文件，不能覆盖旧 HTML。

每份日报必须回答：

1. 昨天或前几日的重要判断是否被验证？
2. 是否有信息错误、数据修正或逻辑需要更正？
3. 哪些主题已经连续多日重要，应该沉淀为核心跟踪事项？
4. 当前结论是否出现重点转向？

## 必备模块

每份日报必须包含以下连续性模块：

```markdown
### 昨日判断回溯

| 昨日判断 | 今日验证 | 结果 | 是否修正 |
|---|---|---|---|
|  |  | 正确 / 部分正确 / 错误 / 待验证 |  |

### 持续重要事项

- 

### 重点转向提醒

- 
```

## 历史文件规则

每份日报都必须有唯一文件路径：

```text
reports/daily/YYYY-MM-DD-HHMM-market-brief.html
```

规则：

- 每天 09:00 推送创建新的当日 09:00 文件。
- 盘中或手动重推也创建新的时间戳文件。
- 旧报告文件不得被新报告覆盖。
- 飞书中发送的链接必须是本次报告唯一链接。
- `latest.html` 或索引页可以指向最新版，但不得用于历史回溯。
- 昨日判断回溯应读取前一份日报的实际文件内容或历史记录，不能从被覆盖的旧链接推断。
- 如果发现过去曾覆盖旧文件，应在下一次日报的“数据修正 / 纠错”里说明历史链接可能不可靠，并从当前起恢复不可变归档。

## 不可省略模块

每日完整日报无论发送到 Codex 线程还是飞书，都不得省略以下两个表格模块：

1. `政策与数据日历`
   - 必须包含日期、事件、已知信息、可能结果、我的结论。
   - 未来事件必须写出高于预期、符合预期、低于预期或偏鹰/偏鸽等情景。

2. `今日核心资产`
   - 必须包含加密、A 股、美股、黄金。
   - 必须包含今日状态、7 日回顾、30 日回顾、未来 1 日、未来 7 日、未来 30 日、未来半年。

如果飞书 text 消息无法良好展示 Markdown 表格，也必须用纯文本表格或分块列表保留完整字段，不能改成简单摘要。

## 回溯检测规则

对过去日报中出现的关键判断必须回测：

- 方向判断：偏多、偏空、震荡、防守。
- 关键价位：是否站回、跌破或突破。
- 宏观判断：CPI、PPI、FOMC、央行讲话是否验证原推断。
- 资金流判断：ETF 流、北向资金、成交额、稳定币供给是否改善或恶化。
- 板块判断：科技、红利、金融、黄金、BTC、ETH 等是否按预期表现。

回溯时必须避免“自圆其说”。如果前次判断错误，应明确写：

```text
昨日判断失效，原因是……
需要修正为……
```

## 持续重要事项沉淀

若同一主题连续 2 日以上影响核心资产，必须进入“持续重要事项”。

示例：

- 美国通胀数据链：CPI、PPI、PCE。
- Fed 利率路径。
- BTC / ETH ETF 资金流。
- 美债收益率和美元。
- A 股政策托底与成交额。
- 中国信用数据、社融和地产政策。
- 黄金实际利率与央行购金。
- 地缘风险和能源价格。

持续事项必须包含：

```text
为什么重要：
目前状态：
对四类核心资产的影响：
下一步触发点：
```

## 重点转向提醒

如果结论从一种状态切换到另一种状态，必须显著提醒。

转向示例：

- 加密：偏空 -> 中性 / 偏多。
- 美股：科技承压 -> 科技修复。
- A 股：防守轮动 -> 政策驱动进攻。
- 黄金：短期受压 -> 利率回落后的反弹确认。

转向提醒必须包含：

```text
原判断：
新判断：
触发原因：
是否确认：
操作含义：
失效条件：
```

## 错误与数据修正规则

如果发现前一日报使用的数据错误、来源质量不足或被后续官方数据修正，必须在下一份日报开头说明。

格式：

```markdown
### 数据修正 / 纠错

- 原信息：
- 新信息：
- 影响：
- 修正后的结论：
```

## 每日任务提示词要求

每日自动报告必须：

1. 读取本线程中过去日报和关键结论。
2. 回溯昨日判断。
3. 下钻最新数据，不把“需要关注”交给用户。
4. 标记持续重要事项。
5. 如果结论转向，优先提醒。
6. 输出完整日报。
