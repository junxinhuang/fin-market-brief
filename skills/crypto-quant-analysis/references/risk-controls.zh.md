# 风控与执行清单

## 仓位与杠杆

- 先定义单笔最大亏损，再决定仓位。
- 杠杆策略必须计算强平价、资金费压力和极端滑点。
- 单币、单交易所、单策略、单方向都要设置上限。
- 山寨币仓位应因流动性、解锁、合约深度和事件风险下调。
- 不使用 martingale、无限补仓、亏损后加倍等结构。

## 分批做空模型

适用于用户已有小空单，并希望抓反弹加仓的情况。

- 底仓：只用于表达方向判断，金额应小到即使止损也不影响心态。
- 反弹加仓：只在价格反弹到压力区并出现失败信号时加，例如放量滞涨、长上影、反抽不过前支撑、OI上升但价格不涨。
- 破位加仓：跌破关键支撑后，等反抽失败再加，不在第一次急跌时追空。
- 禁止加仓：价格横在成本附近、没有确认信号、连续亏损后想摊低成本、funding显示空头拥挤且价格跌不动。
- 每次加仓后必须重算总仓均价、最大亏损、止损价和目标止盈区。
- 加仓后的总风险不能超过预设风险预算；小账户也要按百分比风控，而不是按感觉补仓。

## 交易计划

- 入场条件：价格、信号、成交量、资金费、OI 或事件触发。
- 出场条件：目标、时间止盈、移动止损、信号反转、风险事件。
- 失效条件：关键价位、数据反证、宏观 regime 改变、流动性恶化。
- 暂停条件：连续亏损、异常滑点、交易所故障、数据源失真。
- 复盘条件：每笔交易记录信号、预期、执行、偏差和改进。

## 风险指标

- 最大回撤和回撤持续时间。
- 日内/单周 VaR 或经验分位损失。
- 杠杆后净敞口、方向 beta、BTC beta、相关性。
- Funding burn：资金费连续不利时的损耗。
- Liquidity at risk：按盘口深度估算离场成本。
- Gap risk：周末、公告、黑客、监管、交易所停机。

## 实盘前检查

- 策略在样本外仍有效。
- 成本、滑点、资金费已按保守值测试。
- 使用真实可获得时间的数据。
- 有明确停止交易条件。
- 有交易日志和绩效归因。
- API key 权限最小化，不把提现权限给交易程序。
- 先 paper trade 或极小仓验证执行质量。

## 输出时必须包含

- 风险预算或仓位逻辑。
- 最大可接受亏损。
- 失效条件。
- 需要继续监控的数据。
- 置信度和主要不确定性。
