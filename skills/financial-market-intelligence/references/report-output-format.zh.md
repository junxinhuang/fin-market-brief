# 日报输出格式

## 目标

金融日报的信息密度较高，默认应生成 HTML 完整版，使用咨询报告 / 投研简报风格展示。飞书消息用于提醒、摘要和跳转，不应承载全部复杂表格。

## 默认输出

每次完整日报应同时准备：

1. **HTML 完整版**
   - 用于阅读、归档、截图、打印或导出 PDF。
   - 必须保留政策与数据日历、今日核心资产、回溯、转向提醒、持续事项和资产下钻。

2. **飞书摘要版**
   - 用于快速提醒。
   - 必须包含一句话结论、重点转向、今日核心操作含义和 HTML 报告位置。
   - 若没有可访问的外部链接，应说明完整报告已在 Codex 线程或本地工作区生成。

## HTML 版式

参考模板：

```text
reports/templates/daily-market-brief.html
```

HTML 必须包含以下模块：

- 报告标题、版本、数据截止时间。
- 一句话结论。
- 3-5 个核心驱动。
- 重点转向提醒。
- 昨日判断回溯。
- 政策与数据日历。
- 跨资产仪表盘。
- 今日核心资产。
- A 股专项。
- 美股专项。
- 加密专项。
- 黄金专项。
- 核心资产下钻。
- 机会与风险提醒。
- Watchlist。
- 持续重要事项。
- 操作语言解释。
- 来源和数据缺口说明。

全量模块参考 `benchmark-inspired-modules.zh.md`。在用户明确要求做减法前，日报采用加法版结构。

## 设计规则

- 风格应偏投研 dashboard，不做营销页。
- 优先表格、标签、卡片和短段落。
- 颜色用于表达状态：偏空、偏多、警惕、中性，而不是装饰。
- 表格字段不得因为飞书或移动端展示困难而省略。
- 宽表格必须支持横向滚动。
- HTML 应可直接打开，也应适合后续导出 PDF。

## 飞书推送规则

飞书自定义机器人 Webhook 不适合直接渲染任意 HTML。默认策略：

1. 飞书发送摘要。
2. 摘要中说明完整 HTML 报告位置。
3. 如后续接入飞书文件上传、飞书云文档或对象存储，再发送可点击链接。

如果必须在飞书内完整展示，应改用飞书富文本 `post` 或交互卡片，而不是纯 text。
